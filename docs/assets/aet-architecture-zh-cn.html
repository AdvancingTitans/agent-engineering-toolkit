Agent Engineering Toolkit（AET）— 中文架构全景

Agent 负责完成工作；AET 负责留下证据、守住边界，并让长期改进经过独立评测与人工批准。

最小使用原则

先问你需要证明什么。只担心修改越界就用 Review；只想确认测试真跑过就用 Trace。普通小任务继续使用项目自己的测试和 CI。

状态不混写

命令成功、报告存在、当前仍有效、改动未越界，是四个不同事实。缺证据就保留 UNKNOWN，不靠总分把问题盖住。

无损节省成本

先做确定性预检；可精确续跑和复用；前序目标失败就不打开 Held-out；评测次数由风险和声明决定，而不是固定数字。

权限保持不对称

Agent 可以工作，候选可以参加评测；但候选不能修改裁判、保留题或批准自己上线。Stage 之后仍必须由人明确 Adopt。
