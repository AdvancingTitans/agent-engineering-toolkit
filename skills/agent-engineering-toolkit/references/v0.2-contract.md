# v0.2 Intent Gate contract

Run `aet review --base <revision>` from the repository root. It compares the
current worktree, including untracked files, with `<revision>` and evaluates
`aet.intent.json` (or the path supplied by `--intent`).

The contract is JSON with four required fields:

```json
{
  "intent": "A concise human-readable description of the approved change.",
  "changed_path_budget": 5,
  "allowed_paths": ["aet.intent.json", "src/tool/**", "tests/**"],
  "required_proofs": [{
    "id": "tests",
    "command": "python -m unittest discover -s tests",
    "evidence": ["tests/test_tool.py"]
  }]
}
```

`allowed_paths` accepts exact paths, shell-style filename patterns, and a
directory pattern ending in `/**`. Renames count as both the old and new path.
The intent contract itself is a changed path if it is untracked or different
from the base, so include it in the budget and allowed paths when appropriate.

Each proof needs a unique `id`, a command for the human review record, and one
or more local evidence paths. `aet` verifies only that those evidence paths
exist locally. It never executes commands, so a PASS for `AET-REV-004` means
the proof is declared and its referenced evidence exists—not that the command
has passed.

Rules: `AET-REV-001` validates the contract, `AET-REV-002` enforces the path
budget, `AET-REV-003` checks path scope, and `AET-REV-004` checks proof
evidence. All carry rule version `0.2.0`.
