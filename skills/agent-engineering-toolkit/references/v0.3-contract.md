# v0.3 Evidence Pack and Trace contract

`aet trace --output <trace.json> -- <command> [args...]` executes exactly one
explicit argv without a shell. It records the redacted argv, exit code,
start/finish timestamps, working directory, Git HEAD, worktree digest, and
SHA-256 digests of redacted stdout/stderr artifacts. A non-zero command exit
creates a valid Trace report with execution status `FAIL`; it is never proof of
a successful command. Add repeatable `--redact-pattern <regular-expression>`
options for project-specific secrets. If an argv or log field cannot be safely
decoded or redacted, its status is `UNKNOWN` and its raw value is not stored.

Repeatable `--artifact <relative-path>` captures explicitly declared UTF-8
reports. Each successful capture records redacted-content SHA-256/size plus the
source-file SHA-256/size and `CREATED` or `CHANGED` freshness; the source bytes
are not embedded. Observed scoring accepts Trace routing only through the
runner-injected `./.aet-rollout/bin/aet trace` path. It independently recomputes
the Git snapshot while excluding the Trace JSON itself, rejects `UNKNOWN`, and
requires every argv/intent-declared artifact and stdout/stderr log to match a
real non-link workspace file. Artifact inline content is independently
recomputed with the declared redaction patterns; logs use fixed paths derived
from the Trace output and must be fresh. Outer child argv, Trace argv, and the
intent proof command must match exactly, and proof `evidence` must be an array.
These stricter checks apply to observed Learn
credit; a standalone Trace still records `UNKNOWN` when Git state is unavailable.

`aet evidence pack` validates the supplied JSON report schemas and atomically
writes a host-neutral JSON document. It accepts any subset of `--audit`,
`--review`, and `--trace`; every omitted component is represented as
`UNKNOWN`. Each supplied component records the exact source SHA-256, its
original status summary, and portable report evidence. The Trace section keeps
only execution metadata, redacted excerpts, and artifact digests—never raw
stdout or stderr logs.

Audit and review do not execute commands. Evidence Pack compilation does not
contact MCP servers, model APIs, or any remote service.
