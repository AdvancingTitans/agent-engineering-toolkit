# AET v1 product contract

Use four independent surfaces: `audit` is static hygiene, `review` is a human-authored diff boundary, `trace` is explicit execution evidence, and `evolve` is cited repository archaeology. A report cannot imply a missing surface passed.

Evidence is L0 declared, L1 local static, L2 executed Trace, L3 local Git, L4 explicit remote retrieval, or L5 human attestation. Model output is never an evidence level or hard gate. `aet triage` ranks remediation with visible factors only; it does not alter status.

Before claiming completion, attach the relevant JSON artifact and state remaining `UNKNOWN` items. For release work, run strict audit, intent review, bound successful proof trace, Evidence Pack, package install smoke, and record the Git tag/release URL.
